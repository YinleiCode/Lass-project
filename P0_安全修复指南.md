## Bug #2(优先修):云函数无身份校验

### 严重程度:🔴 高

任何获得 openid 的人都可以调用任何云函数。意味着:
- 家长能调用 `createOrder`(伪造缴费记录)
- 家长能调用 `checkin`(篡改考勤数据)
- 任何陌生用户能调用 `students.list`(获取全部学员手机号)

### 修复方案

#### 步骤 1:创建鉴权工具函数

在每个需要保护的云函数目录中,创建 `auth.js`(或在共享 utils 中):

```javascript
// cloudfunctions/[函数名]/auth.js
const cloud = require('wx-server-sdk')

/**
 * 校验调用者是否为已注册老师
 * @returns {Promise<{openid: string, teacher: object}>}
 * @throws 如果不是老师,抛出 401 错误
 */
async function requireTeacher() {
  const { OPENID } = cloud.getWXContext()
  if (!OPENID) {
    throw { code: 401, message: '未登录' }
  }

  const db = cloud.database()
  const result = await db.collection('teachers')
    .where({ openid: OPENID })
    .limit(1)
    .get()

  if (result.data.length === 0) {
    throw { code: 403, message: '无权限:仅老师可调用' }
  }

  return { openid: OPENID, teacher: result.data[0] }
}

/**
 * 校验调用者是否为家长,且必须是指定学员的家长
 * @param {string} studentId
 */
async function requireParentOf(studentId) {
  const { OPENID } = cloud.getWXContext()
  if (!OPENID) {
    throw { code: 401, message: '未登录' }
  }

  const db = cloud.database()
  const result = await db.collection('students')
    .where({ _id: studentId, parent_openid: OPENID })
    .limit(1)
    .get()

  if (result.data.length === 0) {
    throw { code: 403, message: '无权限:仅该学员家长可调用' }
  }

  return { openid: OPENID, student: result.data[0] }
}

/**
 * 校验调用者已登录(老师或家长均可)
 */
async function requireAnyUser() {
  const { OPENID } = cloud.getWXContext()
  if (!OPENID) {
    throw { code: 401, message: '未登录' }
  }
  return { openid: OPENID }
}

module.exports = { requireTeacher, requireParentOf, requireAnyUser }
```

#### 步骤 2:每个云函数顶部加上身份校验

按下表给每个云函数加校验:

| 云函数 | 校验类型 | 说明 |
|--------|---------|------|
| `cm_initDB` | requireTeacher | 仅老师可初始化数据库 |
| `cm_batchBalance` | requireTeacher | 老师才能批量查余额 |
| `cm_checkin` (点名) | requireTeacher | 仅老师能点名 |
| `cm_createOrder` (缴费) | requireTeacher | 仅老师能登记缴费 |
| `cm_createSchedule` (排课) | requireTeacher | 仅老师能排课 |
| `cm_courseReminder` (提醒) | requireTeacher 或定时触发跳过校验 | 定时触发器调用时 OPENID 为空 |
| `cm_bindParent` | requireAnyUser | 任何登录用户都能绑定 |
| `cm_getMyChild` (家长查孩子) | requireParentOf | 必须是该学员的家长 |
| `cm_submitLeave` (家长请假) | requireParentOf | 必须是该学员的家长 |

⚠️ 真实云函数名可能与上表不一致。**请先列出 `cloudfunctions/` 目录,把每个云函数的真实名字对应到上表的"校验类型",再开始改**。

#### 步骤 3:修改云函数代码

每个云函数的修改模板:

```javascript
// cloudfunctions/cm_xxx/index.js
const cloud = require('wx-server-sdk')
const { requireTeacher } = require('./auth')  // 或 requireParentOf 等

cloud.init({ env: cloud.DYNAMIC_CURRENT_ENV })

exports.main = async (event, context) => {
  // ⭐ 在最前面加校验
  try {
    const { openid, teacher } = await requireTeacher()
    // 原有业务逻辑...
    
  } catch (err) {
    if (err.code === 401 || err.code === 403) {
      return { success: false, error: err.message, code: err.code }
    }
    console.error('云函数执行失败:', err)
    return { success: false, error: err.message || '内部错误' }
  }
}
```

#### 步骤 4:定时触发器特殊处理

`cm_courseReminder` 如果是被定时触发器调用的,`OPENID` 为空。需要这样处理:

```javascript
exports.main = async (event, context) => {
  // 定时触发器调用时 event.source === 'timer-trigger' 或类似标记
  const isTimerTrigger = !cloud.getWXContext().OPENID
  
  if (!isTimerTrigger) {
    await requireTeacher()
  }
  // ... 业务逻辑
}
```

#### 验收方式

修完后做这两个测试:

1. **正向测试**:用老师 openid 调用 `cm_checkin`,应正常返回
2. **负向测试**:在小程序里用一个普通用户 openid(没注册成老师的)调用 `cm_checkin`,应返回 `{ success: false, code: 403 }`

## Bug #1:老师注册无身份验证

### 严重程度:🔴 高

`identity.js` 直接从客户端写入 `teachers` 集合 → 任何人扫码都能把自己注册成老师。

### 修复方案

#### 方案选择(按合适程度排序)

**方案 A(推荐):邀请码机制**

- 老师本人在小程序设置中生成邀请码
- 新老师注册必须输入邀请码才能创建 `teachers` 记录
- 适合"工作室"场景:你 + 朋友 + 1-2 个助教

**方案 B:固定老师不开放注册**

- 50人规模、自用场景,不需要"开放注册"
- 第一位老师由 `cm_initDB` 创建时直接 hardcode openid 写入
- 后续如需添加老师,改云函数代码或操作云数据库控制台

**方案 C:手机号白名单**

- 在云数据库 `config` 集合预先写入"允许注册的手机号列表"
- 注册时校验手机号是否在白名单
- 适合机构场景

我**推荐方案 B**,因为:
- 你和朋友只需要 1-2 个老师账号
- 实现最简单,代码量最少
- 不容易出错

如果用户偏好方案 A 或 C,告诉我们再切换方案。

#### 方案 B 的具体实现

##### 1. 改 cm_initDB 云函数

```javascript
// cloudfunctions/cm_initDB/index.js
exports.main = async (event, context) => {
  const { OPENID } = cloud.getWXContext()
  const db = cloud.database()
  
  // 创建9个集合(已有逻辑)
  // ...
  
  // ⭐ 把当前调用者注册为第一位老师
  const existing = await db.collection('teachers')
    .where({ openid: OPENID })
    .limit(1)
    .get()
  
  if (existing.data.length === 0) {
    await db.collection('teachers').add({
      data: {
        openid: OPENID,
        name: event.teacherName || '王老师',
        org_name: event.orgName || '颂恩艺术',
        created_at: db.serverDate(),
        is_admin: true  // 第一位老师为管理员

    })
  }
  
  return { success: true, message: '初始化完成' }
}
```

## Bug #3:parent/home 加载时空白页

### 严重程度:🟡 中

家长首次打开小程序,加载数据期间页面完全空白(因为 `loading=true` 时 `wx:else` 不渲染),体验差,可能让用户以为程序坏了。

### 修复方案

#### 步骤 1:修改 home.wxml

把现有结构:

```xml
<!-- 假设当前是这样 -->
<view wx:if="{{loading}}">加载中...</view>
<view wx:else>
  <!-- 大量内容 -->
</view>
```

改为:

```xml
<view class="page-container">
  <!-- 加载状态:用骨架屏代替单纯文字 -->
  <block wx:if="{{loading}}">
    <view class="skeleton">
      <view class="skeleton-hero"></view>
      <view class="skeleton-card"></view>
      <view class="skeleton-card"></view>
    </view>
  </block>
  
  <!-- 错误状态 -->
  <block wx:elif="{{error}}">
    <view class="empty-state">
      <text class="empty-text">加载失败 😢</text>
      <button class="retry-btn" bindtap="reload">重试</button>
    </view>
  </block>
  
  <!-- 正常内容 -->
  <block wx:else>
    <!-- 原有的大量内容 -->
  </block>
</view>
```

#### 步骤 2:home.js 加上错误处理

```javascript
Page({
  data: {
    loading: true,
    error: false,
    // ... 其他数据
  },
  
  async onLoad() {
    await this.reload()
  },
  
  async reload() {
    this.setData({ loading: true, error: false })
    try {
      const data = await api.getMyChildOverview()
      this.setData({
        loading: false,
        ...data
      })
    } catch (err) {
      console.error('加载家长首页失败:', err)
      this.setData({ loading: false, error: true })
      wx.showToast({ title: '加载失败,请重试', icon: 'none' })
    }
  }
})
```
