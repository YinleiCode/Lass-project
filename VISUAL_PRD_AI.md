# 课程管理小程序视觉设计 PRD

## 1. 项目定位

本项目是一个面向播音主持培训机构 / 小型工作室的微信小程序，用于替代老师通过 Excel、记事本、微信群管理课程的低效方式。

产品核心目标：让老师高效管理学员、课时、缴费、排课、点名和课后反馈；让家长清楚查看孩子剩余课时、上课记录、课后反馈和缴费记录。

设计稿目标：生成一套适合微信小程序落地的高保真视觉设计稿，覆盖老师端和家长端核心页面。

## 2. 用户角色

### 老师 / 管理员

主要使用场景：日常排课、点名、扣课时、登记缴费、查看经营数据、管理学员资料。

核心诉求：效率高、信息清楚、操作步骤少、避免漏扣课时和漏反馈。

### 家长

主要使用场景：查看孩子剩余课时、上课记录、课后反馈、缴费记录、提交请假。

核心诉求：安心、透明、能快速知道孩子学习情况和课时余额。

## 3. 设计风格要求

整体风格：专业、亲和、轻教育感，不要过度卡通。

视觉关键词：清晰、温暖、可信赖、高效、轻量。

推荐方向：浅色背景、柔和蓝色作为主色，搭配少量暖色提示重要状态。

避免方向：金融后台感、复杂企业 SaaS 感、强儿童卡通风、过度渐变和装饰。

适配要求：优先移动端微信小程序，页面宽度按 iPhone 主流尺寸设计，注意底部 tabBar 和安全区。

## 5. 核心页面清单

设计稿至少覆盖以下页面。

### 5.1 身份选择页

用途：首次进入时选择老师或家长身份。

关键内容：产品名称、角色入口卡片、简短说明。

设计重点：入口清晰，老师和家长角色区分明显。

### 5.2 老师首页

用途：老师查看今日待上、本月收入、本月课消、低课时提醒。

关键模块：数据概览卡片、今日课程列表、低课时预警、快捷操作。

关键操作：去排课、去点名、添加学员、登记缴费。

设计重点：数据一眼可读，今日任务优先级最高。

### 5.3 学员列表页

用途：老师查看和筛选所有学员。

关键模块：搜索框、标签筛选、学员卡片、剩余课时、状态。

关键操作：新增学员、进入学员详情、长按操作。

设计重点：列表密度适中，剩余课时和预警状态醒目。

### 5.4 学员详情页

用途：查看单个学员的资料、课时、课程记录、缴费记录。

关键模块：学员基础信息、剩余课时卡片、近期课程、缴费记录、反馈记录。

关键操作：编辑资料、登记缴费、排课、删除或归档。

设计重点：信息分区明确，老师能快速判断是否需要续费或跟进。

### 5.5 新增 / 编辑学员页

用途：录入或修改学员资料。

关键字段：学员姓名、家长姓名、家长手机号、标签、备注。

设计重点：表单简洁，必填项清晰，适合单手操作。

### 5.6 缴费登记页

用途：老师为学员登记缴费并增加课时。

关键字段：学员、课程包、课时数、金额、支付方式、缴费日期、备注。

关键反馈：提交前确认弹窗，提交后成功提示。

设计重点：金额和课时要突出，减少误操作。

### 5.7 课表页

用途：老师按周查看课程并创建排课。

关键模块：周切换、每日课程列表、课程状态、创建排课弹窗。

关键操作：上一周、下一周、回到今天、新增排课、进入点名。

设计重点：日历结构清晰，课程状态明显区分。

### 5.8 创建排课弹窗 / 页面

用途：创建单次课或周期课。

关键字段：课程类型、学员、课程包、日期、时间、教室、周期截止日期。

设计重点：单次课和周期课切换清楚，提交前能看出实际排课时间。

### 5.9 点名页

用途：老师对某节课学员点名并扣课时。

关键模块：课程信息、学员点名列表、到课/请假/缺席状态、扣课时数、提交按钮。

关键操作：提交点名、撤销点名。

设计重点：避免误点名；扣课时数字要清楚；提交按钮状态明确。

### 5.10 课后反馈页

用途：老师为学员填写课后反馈。

关键模块：评分维度、快捷评语模板、文本评语、语音反馈上传。

关键操作：保存反馈、切换学员。

设计重点：填写负担低，评分和文字输入体验顺畅。

### 5.11 家长首页

用途：家长查看孩子课程概况。

关键模块：孩子信息、剩余课时、下次上课、最新反馈、快捷入口。

关键操作：查看记录、查看缴费、提交请假、查看反馈详情。

设计重点：安心感和透明感，剩余课时与下次课程突出。

### 5.12 家长记录页

用途：家长查看上课记录和反馈列表。

关键模块：月份筛选、反馈卡片、上课日期、评分摘要、老师评语摘要。

设计重点：时间线清楚，家长能快速看到学习进展。

### 5.13 反馈详情页

用途：家长查看单节课详细反馈。

关键模块：课程信息、评分维度、老师评语、语音反馈。

设计重点：内容阅读舒适，评分表达直观。

### 5.14 缴费记录页

用途：家长查看历史缴费和课时购买情况。

关键模块：缴费列表、金额、购买课时、支付方式、日期。

设计重点：财务信息清楚可信。

### 5.15 请假页

用途：家长选择未来课程并提交请假。

关键字段：课程、请假原因、备注。

设计重点：流程简单，提交结果明确。

### 5.16 个人中心页

用途：老师或家长查看个人信息和常用设置。

老师侧关键内容：老师信息、课程包管理、邀请码入口、系统操作。

家长侧关键内容：绑定孩子、孩子资料、联系客服。

设计重点：功能收纳清晰，避免首页过载。

## 6. 关键业务流程

### 老师日常上课流程

1. 老师进入首页查看今日课程。
2. 点击课程进入点名页。
3. 对学员设置到课 / 请假 / 缺席。
4. 提交点名后自动扣课时。
5. 进入课后反馈页填写评分和评语。
6. 家长端可查看反馈。

### 老师新增学员并收费流程

1. 老师进入学员页。
2. 点击新增学员。
3. 填写学员和家长信息。
4. 进入缴费登记。
5. 选择课程包、课时数、金额并提交。
6. 学员余额增加。

### 家长查看反馈流程

1. 家长进入首页。
2. 查看最新反馈卡片。
3. 点击进入反馈详情。
4. 查看评分、评语和语音反馈。

### 家长请假流程

1. 家长进入请假页。
2. 选择未来课程。
3. 填写原因和备注。
4. 提交请假。

## 7. 关键状态与视觉表达

课程状态：待上、已点名、已取消。

点名状态：到课、请假、缺席。

课时状态：正常、低课时、已用完。

缴费状态：已登记。

反馈状态：未填写、已填写。

低课时预警建议使用醒目的暖色，但不要造成强烈警报感。

成功状态建议使用柔和绿色。

普通信息状态使用中性色。

## 10. 视觉注意事项

页面应适合微信小程序原生实现，避免复杂不可落地的动效和超重视觉结构。

老师端强调效率和信息密度；家长端强调安心、清楚和可读性。

不要设计在线支付、聊天、直播、作业批改、积分系统、多校区、多权限后台等 V2 功能。

不要把产品设计成大型 SaaS 后台；它是小型培训工作室日常高频使用的小程序。
